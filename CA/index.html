Learning Resource: Internet and World Wide Web

Introduction To Web Architecture

Web architecture is the design and layout of a website.
A web server is an application installed on a physical or virtual server that uses HTTP to serve the files that make up web pages to users browsers in response to their requests.
A server can be physical or virtual, physical servers are placed in racks.

Web Protocols

HTTP:

(Hyper Text Protocol) Defines rules for requesting and transferring web resources.

HTTPS:

This is the secure version of HTTP it uses encrytption(ssl/tls) to protect data.

TCP/IP:

When data is sent over the interent protocol called Tranmission Control Protocol is used. TCP takes care of communication between your application software and your network software. it is responsible for breaking down data into IP packets before they are sent and assembling them when they arrive

FTP:

File Transfer Protocol enables transfer of files such as your web files from your local machine up on to a web browser.

Web Concepts

URLs and DNs

URL stands for uniform resource locators
DNS stands for domain name system. it translates human readable domain names into IP addresses, allowing browsers to locate web servers by name rather than numerical IP addresses.

TLD:

This is the last segment of your domain name, e.g. :

.ie
.com
.net
.info

Cookies and browsers

Cookies ar small pieces of data sent by web servers to a clients broswer, used for users authentication, session management and tracking user behavior.
